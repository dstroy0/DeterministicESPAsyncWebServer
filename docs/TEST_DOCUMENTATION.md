# Test Documentation

DeterministicESPAsyncWebServer is tested with **Unity** suites that run on a
native x86/x64 host - no ESP32 hardware required. The latest pass/fail results
and per-test tables are in the auto-generated [TEST_REPORT.md](TEST_REPORT.md)
(regenerated by the *Test Report* GitHub workflow on every push to `main`).

## Running the suites

```bash
pio test -e native -e native_app -e native_ssh \
         -e native_ssh_hardened -e native_ssh_conn -e native_compliance
```

Or regenerate the full report locally:

```bash
bash test/run_tests.sh   # writes docs/TEST_REPORT.md
```

## Environments

Each PlatformIO environment compiles a specific slice of the stack so that
features (and compile-time flags) are exercised in isolation:

| Environment           | Purpose                                                                       |
| --------------------- | ----------------------------------------------------------------------------- |
| `native`              | Transport, session, presentation, HTTP parser, WebSocket, SSE                 |
| `native_app`          | Full Layer-7 server: auth, file serving, multipart, method dispatch           |
| `native_ssh`          | SSH crypto + protocol stack (software crypto paths, no mbedTLS)               |
| `native_ssh_hardened` | SSH built with `DETWS_SSH_ALLOW_PASSWORD=0` (publickey-only)                  |
| `native_ssh_conn`     | SSH wired through the real transport/session byte-pump (`DETWS_ENABLE_SSH=1`) |
| `native_compliance`   | Strict-defaults build (`DETWS_ENFORCE_HOST_HEADER=1`) for Host/Content-Length |

The `native` and `native_app` environments build with
`DETWS_ENFORCE_HOST_HEADER=0` because their fixtures predate the RFC 7230 §5.4
Host rule and exercise parser mechanics that are orthogonal to Host policy; the
strict behavior is verified separately in `native_compliance`.

## Known-answer test vectors

Crypto suites use genuine external vectors, not self-generated ones:

- SHA-256 / HMAC-SHA2-256 - NIST FIPS 180-4 and RFC 4231 vectors (cross-checked
  with `sha256sum` / `openssl`).
- AES-256-CTR - NIST SP 800-38A vectors.
- RSA-SHA2-256 verification - a real signature produced with
  `openssl dgst -sha256 -sign` over an RSA-2048 key; the publickey-auth test uses
  a real signature over the exact RFC 4252 §7 signed-data blob.
